# プロジェクトビジョン・目的

## このプロジェクトが目指すこと

**Claude Code（VSCode上のAI）を主インターフェースとして、会社テンプレートから
高品質な提案書PPTXを対話的に生成・管理するシステムを構築する。**

## なぜこのアプローチか

### 従来の課題
- 提案書作成に数時間かかる（内容思考 + デザイン作業の混在）
- テンプレートの使い方が人によってバラバラ
- 過去の資料が個人PCに埋もれて再利用されない

### このシステムの解決策
- 「話しかけるだけ」で会社テンプレートに沿ったPPTXが生成される
- 中間言語（JSON）で内容とデザインを分離 → AI が内容に集中できる
- GitHub で生成物を一元管理 → ナレッジの蓄積・再利用が可能

## アーキテクチャの考え方（プロッタ―・パラダイム）

```
[思考層] Claude Code がユーザーと対話 → JSON アウトライン生成
    ↓
[中間言語] JSON（スライド構造・オブジェクト・画像プロンプトを含む）
    ↓
[実行層] pptx_engine.py が python-pptx で忠実に描画
    ↓
[画像層] Gemini API（NANOBANAPRO）が各スライドに AI 画像生成
    ↓
[管理層] GitHub に PPTX + レシピ JSON を自動コミット・蓄積
```

思考（AI）と実行（Python）を分離することで：
- AI の出力を検証・修正してから実行できる
- 実行ロジックを別の AI に差し替えられる
- 中間言語 JSON が「設計図」として再利用できる

## 今後の拡張方向（優先度順）

1. **レシピライブラリの充実** — 業種・用途別のテンプレート集
2. **視覚的セルフコレクションの自動化** — 生成→サムネイル確認→修正ループ
3. **章扉・表紙の AI 画像** — `chapter_photo` + `position: "auto"` の活用
4. **マルチテンプレート対応** — 会社以外のテンプレートへの対応
5. **スライド単位の差し替え** — 既存 PPTX の特定スライドだけ更新
