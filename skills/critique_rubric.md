# スキル: スライド診断ルーブリック

> **使い方**: `--thumbnail` で生成したPNGを評価する際に読む。通常の生成作業では不要。

---

## 診断チェックリスト（各スライドを確認）

### A. タイポグラフィ（文字）
- [ ] タイトルは25〜35文字か
- [ ] キーメッセージは40〜70文字か
- [ ] 箇条書きは1項目30〜50文字・3〜5項目か
- [ ] テキストがプレースホルダーからはみ出していないか

### B. カラー
- [ ] 使用色が3色以内か
- [ ] 赤（課題）と青（提案）の対比が適切か
- [ ] 白文字が濃い背景の上にあるか（コントラスト）

### C. レイアウトバランス
- [ ] テキスト（左）・図解/画像（右）の配置になっているか
- [ ] 余白が適切か（詰め込みすぎていないか）
- [ ] 図解（objects）または画像（images）が最低1つあるか

### D. 内容品質
- [ ] スライドタイトルだけでメッセージが伝わるか
- [ ] 本文はタイトルの根拠・補足になっているか
- [ ] キーメッセージが「動詞終止」または「体言止め」か

### E. 図解品質（objects がある場合）
- [ ] 現状→提案の対比になっているか
- [ ] 矢印の方向が論理の流れと合っているか
- [ ] ボックスのサイズが揃っているか（left/widthの計算）

---

## 診断レポートの出力形式

```
## スライド診断レポート

### スライド X: [タイトル]
- **総合評価**: ★★★★☆ (4/5)
- **良い点**: キーメッセージが明確。図解が効果的
- **改善提案**:
  - 本文の箇条書き6項目→4項目に絞る
  - 画像を右側（left:7.5）に移動

### 全体コメント
- 色使いは統一されている
- chapter スライドをすべて chapter_photo に変更すると視覚的インパクトが増す
```

---

## 優先度付き改善フロー

1. **致命的（即修正）**: テキストはみ出し、色が見えない
2. **重要（次回修正）**: 1スライド複数メッセージ、図解なし
3. **推奨（余裕があれば）**: 画像位置、文字数微調整

---

## スライドタイプ別評価基準

### contentスライドの理想スコア
| 要素 | あり | なし |
|---|---|---|
| title | 必須 | -2点 |
| subtitle（キーメッセージ） | +1点 | 0点 |
| body（本文） | 必須 | -1点 |
| objects（図解） | +1点 | 0点 |
| images（画像） | +1点 | 0点 |

5点満点を目指す。3点以下は要改善。
